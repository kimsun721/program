Next.js — App Router

• middleware.ts 가 /admin/**·/instructor/**·/my/** 경로의 JWT를 검사

• 인덱스: (courseId, status), (userId)

수료증 PDF

• @react-pdf/renderer로 서버 렌더링

04

인증 & RBAC 흐름

3중 권한 가드

① Middleware — 경로 prefix 매칭, 비로그인은 /login?next=, 권한 부족은 /403.
② requireRole() — 페이지(RSC) 진입 시점에서 세션 + role 재검증.
③ assertRoleInAction() — 모든 mutation 액션은 액션 첫 줄에서 검증.

한 명이 여러 역할

users.role은 배열(String[]).
예) 학생이 강사 신청 후 승인되면 ["STUDENT", "INSTRUCTOR"]가 됨.
ADMIN은 시드에서만 부여.

07

강사(INSTRUCTOR) 영역

강사 대시보드

KPI 4종 — 총 강의 / 총 수강생 / 평균 평점 / 미답 Q&A.
최근 강의 5개 테이블, 미답 Q&A가 0이 아니면 카드가 amber로 강조.

강의 CRUD

제목·slug·설명·썸네일·언어·카테고리·레벨·가격.
PUBLISHED 강의는 수정 불가(검증 우선). DRAFT 강의만 삭제 가능.

커리큘럼 에디터

섹션 추가/이름 변경/삭제 + 차시 추가/수정/삭제. 클라이언트 컴포넌트는 useTransition으로 펜딩 상태 표현.
차시 추가 시 sortOrder는 MAX + 1로 자동 부여. 게시 요청 직전 섹션 0개면 서버에서 거절.

Q&A 답변 페이지

본인 강의 질문만 노출. 답변 작성 시 QnaQuestion.status를 ANSWERED로 전이.

본인 강의 가드

모든 액션의 첫 줄에서 course.instructorId == 내 profile.id 검증.
타인 강의 ID로 호출해도 "본인 강의가 아닙니다" 로 거절.

08

관리자(ADMIN) 영역

운영 대시보드

전체 회원·강의·수강 건수·평균 평점, 강사 신청 대기·강의 승인 대기.
최근 가입 회원/등록 강의 5개씩 노출. 대기 카운트가 1 이상이면 amber 강조.

회원 관리

이메일/닉네임 검색, 상태 필터.
상태 변경: ACTIVESUSPENDEDDELETED. ADMIN 계정은 변경 차단.

강사 승인

PENDING/APPROVED/REJECTED/ALL 탭. 자기소개·경력 미리 보고 승인 또는 5자 이상 사유로 반려.
승인 시 트랜잭션으로 profile.status 갱신 + users.role에 INSTRUCTOR 추가.

강의 승인

REVIEW 강의를 카드로 검토. 미리보기 버튼으로 학생 시점 새 탭 오픈.
반려 시 사유 저장 → 강사가 그대로 확인.

카테고리·언어 운영

카테고리 CRUD(이름·슬러그 중복 방지) + 카테고리/언어 활성·비활성 토글.
비활성 항목은 강사의 강의 등록 폼에 노출되지 않아 콘텐츠 품질을 유지.

09

학생(STUDENT) 학습 강화 기능

Q&A

강의 상세 페이지에서 작성, 차시 단위로 묶을 수 있음. 작성·수정·삭제 본인만.

단어장

강의별 단어장 생성. 단어·뜻·예문, 학습 완료 토글, 진행률 바 표시.

학습 노트

차시별 메모. 영상 시각(초) 첨부 가능. 학습 페이지 내 인라인 작성 위젯 제공.

수료증 (PDF)

진도 100% 자동 노출 → 1클릭 발급. S/N 부여, A4 가로 PDF로 즉시 다운로드.

구현 포인트

① 모든 액션은 "본인 소유" + "수강 중" 이중 검증
② 단어장은 강의 미연결도 허용(개인 범용 단어장)
③ 노트의 영상 시각은 정수만 허용, 학생이 임의 큰 값 넣어도 무시되지 않게 검증
④ 수료증 PDF는 @react-pdf/renderer의 renderToBuffer로 서버 응답, 캐싱 없이 inline 표시

End-to-End 시나리오 점검

① 학생 시나리오

가입 → 이메일 인증 → 로그인 → 강의 탐색 → 수강 → 차시 시청(진도 자동 저장) → Q&A 작성 → 강사 답변 확인 → 진도 100% → 수료증 발급 → PDF 다운로드.

② 강사 시나리오

학생 가입 → 강사 신청 → 관리자 승인 → 세션 새로고침 시 강사 메뉴 노출 → 강의 생성(DRAFT) → 섹션·차시 추가 → 게시 요청 → 승인 후 PUBLISHED → 수강생 Q&A 답변.

③ 관리자 시나리오

ADMIN 로그인 → 대시보드 KPI 확인 → 강사 신청 PENDING 처리 → 강의 REVIEW 승인/반려 → 부적절 회원 SUSPENDED → 카테고리 추가/비활성.

④ 권한 침입 테스트

학생 계정으로 /admin/dashboard·/instructor/* 직접 접근 → 미들웨어가 /403로 리다이렉트.
액션 호출 시 임의 ID 주입해도 본인 소유 검증으로 차단.

회고 & P3 예고

잘 된 점

• 미들웨어 + RSC 가드 + 서버 액션 가드 3중 방어로 권한 누락이 거의 발생하지 않음.
• role을 배열로 둬서 "학생인 강사"라는 자연스러운 다중 역할이 그대로 표현됨.
• 강의 게시 워크플로우(DRAFT→REVIEW→PUBLISHED/HIDDEN)가 운영 품질 기준을 강제함.

아쉬운 점 / 트레이드오프

• 차시 순서 변경은 추가/삭제만 지원 (드래그앤드롭 보류).
• 수료증 PDF는 한글 폰트 임베드 미적용 (영문 템플릿).
• Q&A는 단일 깊이 답변만 — 대댓글/Markdown은 P3 이월.

P3 — 운영 단계 예고

결제(PortOne v2 단건/구독), 환불·정산, 알림(이메일/푸시) 스케줄링·배치, 모니터링·로깅(Sentry + 구조화 로깅), AI 회화/튜터 1:1 화상 수업, 그리고 기존 코드 리팩토링·성능 최적화.
