## 1. P2 단계 목표

P1은 수강생(STUDENT) 입장의 강의 탐색·수강·리뷰 기능에 집중했다. P2에서는 플랫폼이 다중 역할 체계로 동작하도록 확장한다.

| 목표 | 내용 |
|------|------|
| 역할 분리 | STUDENT / INSTRUCTOR / ADMIN 3개 역할 분리, 역할별 라우트·권한·UI 분리 |
| 인증 강화 | NextAuth 세션에 role 반영, 미들웨어 기반 RBAC, 강사 승인 워크플로우 |
| 강사(선생님) 기능 | 강의·섹션·차시 CRUD, 강사 자기 강의 통계, Q&A 답변 |
| 관리자 기능 | 회원/강사 승인·차단, 강의 게시 승인, 카테고리·언어 운영 |
| 수강생 추가 기능 | Q&A 질문, 단어장, 학습 노트, 수강 완료 시 수료증 |

---

## 2. 역할 정의

| 역할 | 코드 | 가입 경로 | 주요 권한 |
|------|------|-----------|-----------|
| 학생 | `STUDENT` | 일반 회원가입(이메일 인증) | 강의 탐색·수강, 리뷰, Q&A 작성, 단어장·노트, 수료증 |
| 선생님 | `INSTRUCTOR` | 학생 가입 후 "강사 신청" → 관리자 승인 | 본인 강의 CRUD, 본인 강의 Q&A 답변, 수강생 통계 |
| 관리자 | `ADMIN` | 시스템 시드(seed)로만 부여 | 회원/강사 관리, 강의 승인, 카테고리·언어 운영, 대시보드 |

> 한 사용자는 복수 역할을 가질 수 있다 (예: 학생이면서 강사). DB의 `role`은 배열로 유지한다.

## 3. 주요 기능 (P2 추가/확장)

### 3-1. 인증 & 권한

| 기능 | 내용 |
|------|------|
| 역할 기반 라우팅 | `middleware.ts`에서 경로(prefix)별 role 검증 |
| 강사 신청 | 학생이 강사 프로필(실명·경력·자기소개) 작성 후 신청 |
| 강사 승인 | 관리자가 `PENDING` → `APPROVED`/`REJECTED` 처리 |
| 비밀번호 변경 | 마이페이지 내 현재 비밀번호 확인 후 변경 |

### 3-2. 선생님 (INSTRUCTOR)

| 기능 | 내용 |
|------|------|
| 강사 대시보드 | 본인 강의 수, 총 수강생, 평균 평점, 미답 Q&A 카운트 |
| 강의 CRUD | 제목·설명·언어·카테고리·레벨·가격·썸네일 등록/수정/삭제(DRAFT) |
| 섹션/차시 관리 | 섹션 추가·순서 변경, 차시 추가(영상 URL, 시간, 미리보기 여부) |
| 게시 요청 | DRAFT → REVIEW 전환, 관리자 승인 시 PUBLISHED |
| Q&A 답변 | 내 강의에 달린 학생 질문에 답변 |

### 3-3. 관리자 (ADMIN)

| 기능 | 내용 |
|------|------|
| 운영 대시보드 | 총 회원/강의/수강/평균 평점, 최근 가입자·강의 등록 |
| 회원 관리 | 회원 목록·검색, 정지(SUSPENDED)/해제, 강제 탈퇴 |
| 강사 승인 | 강사 신청 목록, 프로필 확인 후 승인/반려 |
| 강의 승인 | REVIEW 상태 강의 미리보기 후 PUBLISHED/HIDDEN 처리 |
| 카테고리·언어 | 카테고리 CRUD, 언어 활성/비활성 토글 |

### 3-4. 수강생 (STUDENT) 추가

| 기능 | 내용 |
|------|------|
| Q&A | 수강 중인 강의의 차시 단위로 질문 등록, 강사 답변 받기 |
| 단어장 | 강의별 단어장 생성, 단어/뜻/예문 추가, 학습 체크 |
| 학습 노트 | 차시별 메모(타임스탬프 첨부 가능) |
| 수료증 | 강의 진도 100% 시 PDF 수료증 발급 |
