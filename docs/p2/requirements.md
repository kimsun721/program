## 2. 기능 요구사항 (FR)

### 2-1. 인증·계정

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| FR-AUTH-P2-001 | 회원가입 시 기본 role=`STUDENT` 부여 | 필수 |
| FR-AUTH-P2-002 | 세션(JWT)에 `role: string[]` 포함 | 필수 |
| FR-AUTH-P2-003 | 미들웨어로 `/admin/**` → ADMIN, `/instructor/**` → INSTRUCTOR 강제 | 필수 |
| FR-AUTH-P2-004 | 강사 신청(프로필 등록) | 필수 |
| FR-AUTH-P2-005 | 관리자의 강사 승인/반려 | 필수 |
| FR-AUTH-P2-006 | 비밀번호 변경(현재 비밀번호 확인) | 필수 |
| FR-AUTH-P2-007 | 계정 정지(SUSPENDED) 시 로그인 차단 | 필수 |

### 2-2. 강사(INSTRUCTOR)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| FR-INST-001 | 강사 대시보드(요약 통계) | 필수 |
| FR-INST-002 | 본인 강의 목록·생성·수정·삭제(DRAFT만 삭제) | 필수 |
| FR-INST-003 | 섹션·차시 CRUD, 순서 변경 | 필수 |
| FR-INST-004 | 강의 게시 요청(DRAFT → REVIEW) | 필수 |
| FR-INST-005 | 본인 강의 Q&A 목록 및 답변 작성 | 필수 |
| FR-INST-006 | 본인 강의 수강생 통계(인원·진도 평균) | 권장 |

### 2-3. 관리자(ADMIN)

| ID | 요구사항 | 우선순위 |

| FR-ADMIN-001 | 운영 대시보드(요약 KPI) | 필수 |

### 2-4. 학생(STUDENT) 추가

## 3. 비기능 요구사항 (NFR)

| ID | 요구사항 | 수용 수준 |
|----|----------|-----------|
| NFR-P2-001 | 권한 위반 시 403 페이지 노출, 서버 로그 기록 | 필수 |
| NFR-P2-002 | 강사/관리자 라우트는 SSR 단계에서 권한 검증(클라이언트만 X) | 필수 |
| NFR-P2-003 | Q&A·노트 등 텍스트 입력은 길이 제한 + XSS 방지 | 필수 |
| NFR-P2-004 | 강사 대시보드 통계는 N+1 회피, 단일 쿼리 | 권장 |

## 4. 제약 / 가정

- 결제·구독·정산은 P3에서 다루므로 강의 구매는 P1 수동 수강신청 흐름을 유지.
- 영상 업로드 자체는 P3 범위이며, P2에서는 HLS URL을 강사가 직접 입력하는 형태.
- 알림(이메일·푸시)은 P3에서 도입. P2에서는 화면 노출만으로 충분.
- 한 사용자가 다중 역할을 가질 수 있다 (STUDENT + INSTRUCTOR 등).
